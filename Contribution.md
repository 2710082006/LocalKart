## Git Branch Strategy

* `main` → Stable production-ready code
* `develop` → Active development branch
* `feature/`client → Frontend development
* `feature/`server → Backend development
* `feature/documentation` → Documentation updates

---

## Collaboration Workflow

1. Create a feature branch from `develop`.
2. Implement assigned tasks.
3. Commit changes with meaningful commit messages.
4. Push changes to GitHub.
5. Create a Pull Request (PR).
6. Review and merge into `develop`.
7. Merge tested and stable code into `main`.

---

## Expected Contribution Policy

* Every team member will contribute through GitHub commits.
* Tasks will be completed according to assigned responsibilities.
* Code reviews and collaboration will be encouraged throughout development.
* Major project decisions will be discussed collectively by the team.
